# 408 笔记写作规范

## 1. 项目目标

本项目用于持续产出 4 份独立的 408 复习笔记，统一采用 `Markdown` 格式，面向“有一点计算机基础、希望兼顾 408 应试与面试表达”的学习者。

笔记范围包括：

1. 数据结构与算法
2. 操作系统
3. 计算机网络
4. 计算机组成原理

## 2. 已确认约束

- 输出格式：`Markdown`
- 交付方式：四个独立文件
- 单篇目标长度：约 `8000~12000` 字
- 讲解风格：`408 占 70% + 面试占 30%`
- 表达要求：图文并茂、深入浅出、尽量直观
- 配图方式：优先使用 `Mermaid`、表格、流程图、结构图、对比图
- 内容取舍：弱化偏难、冷门、难掌握、实际使用较少的部分
- 受众基础：有一点计算机基础，但不按“已经完全掌握”来默认

## 3. 总体写作原则

### 3.1 内容取舍原则

保留以下内容：

- 高频考点
- 主干知识体系
- 常见题型对应的理解方式
- 面试中高频被问到的核心概念
- 容易混淆、容易丢分、容易说不清的知识点

弱化以下内容：

- 过度偏门的历史细节
- 过于复杂且不利于短期掌握的推导
- 在 408 与常见面试中都不常见的冷门知识点
- 需要大篇幅证明、但性价比较低的理论枝节

### 3.2 表达原则

- 先讲“它是什么”，再讲“为什么”，最后讲“怎么考、怎么答”
- 每章优先给直觉理解，再补规范表述
- 能表格总结的内容尽量表格化
- 能画图的内容尽量画图
- 对易混概念采用“对比式讲法”
- 尽量用“小结论 + 解释 + 例子”的结构

### 3.3 面试兼容原则

每篇不仅服务于做题，还要帮助回答“请你讲讲……”类问题，因此要尽量做到：

- 概念能说清
- 过程能复述
- 对比能讲明白
- 场景能落地
- 常见追问能承接

## 4. 统一结构模板

每一篇笔记都尽量遵循下面的固定框架，以保证后续多轮创作时风格稳定。

```text
# 标题

## 1. 本章学习目标
## 2. 章节导图
## 3. 核心知识展开
## 4. 高频考点总结
## 5. 易错点 / 易混点
## 6. 面试常问
## 7. 刷题与复习建议
## 8. 最后速记版
```

其中：

- “核心知识展开”是主体部分，占全文大头
- “高频考点总结”强调做题视角
- “面试常问”强调口头表达视角
- “最后速记版”强调考前回顾

## 5. 图表规范

### 5.1 图的优先级

1. `Mermaid` 流程图 / 结构图
2. Markdown 表格
3. 简单文本示意图

### 5.2 图表使用场景

- 层次结构：用树状图或流程图
- 数据/概念对比：用表格
- 执行过程：用时序或步骤图
- 模块关系：用结构图

### 5.3 图表风格

- 不追求花哨，追求一眼看懂
- 图要服务于讲解，不单独堆图
- 图后通常补一句“图中最关键的是……”

## 6. 四科统一输出要求

每一篇都必须尽量包含以下模块：

- 学习目标
- 章节导图
- 核心知识详解
- 高频考点
- 易错点 / 易混点
- 面试常问
- 刷题建议
- 速记版总结

## 7. 四科具体范围与取舍

### 7.1 数据结构与算法

定位：以 408 数据结构为主，兼顾常见算法面试套路。

重点保留：

- 时间复杂度、空间复杂度
- 线性表、栈、队列、串、数组
- 树、二叉树、线索树、堆、并查集
- 图的遍历、最小生成树、最短路、拓扑排序、关键路径
- 查找与排序
- 递归、分治、回溯、DFS/BFS、二分、双指针、动态规划入门思维

可弱化：

- 过度繁琐的证明
- 特别冷门的变体结构
- 极少实际使用的复杂变种

### 7.2 操作系统

重点保留：

- 操作系统定义与功能
- 进程、线程、并发、同步互斥
- 调度算法
- 死锁
- 内存管理、分页、分段、虚拟内存
- 文件系统
- 输入输出与设备管理

可弱化：

- 过度细碎的边角机制
- 特定教材里过深的底层实现细节

### 7.3 计算机网络

重点保留：

- 体系结构与分层思想
- 物理层、数据链路层、网络层、传输层、应用层
- TCP/UDP
- IP、ARP、DHCP、NAT
- 路由、拥塞控制、流量控制
- HTTP、HTTPS、DNS

可弱化：

- 过细的报文字段记忆负担
- 实战中极少单独考查的冷门协议细枝末节

### 7.4 计算机组成原理

重点保留：

- 计算机系统层次结构
- 数据表示与运算
- 存储系统
- 指令系统
- CPU 结构与执行过程
- 总线
- 输入输出系统

可弱化：

- 偏难的细碎计算题技巧
- 很少实际使用的复杂微观实现枝节

## 8. 推荐写作顺序

按照“通用重要度 + 对后续理解的支撑作用”，建议顺序为：

1. 数据结构与算法
2. 操作系统
3. 计算机网络
4. 计算机组成原理

## 9. 单篇内部写法约定

### 9.1 段落写法

推荐采用如下微结构：

1. 结论一句话
2. 原理解释
3. 例子或图示
4. 考试怎么考 / 面试怎么问

### 9.2 对比写法

遇到容易混淆的内容，优先使用表格，例如：

- 进程 vs 线程
- TCP vs UDP
- 堆排序 vs 快速排序
- 分页 vs 分段
- SRAM vs DRAM

### 9.3 面试问答写法

面试问题优先采用下面的答题组织方式：

- 先给定义
- 再讲核心机制
- 再讲优缺点或适用场景
- 最后讲与相似概念的区别

## 10. 后续协作约定

后续每次继续创作时，默认遵守本文件，不再重复确认以下内容：

- 使用 Markdown
- 保持四个独立笔记
- 408 占 70%，面试占 30%
- 图表化表达
- 优先讲核心高频
- 弱化低频难点

如果后续用户提出新的偏好，以用户最新要求为准。
