# 异步、并发、线程与任务

## C++ 与 C# 并发心智差异

C++ 中你经常直接面对：

- `std::thread`
- `std::mutex`
- `std::atomic`
- condition variable
- coroutine
- 线程池库或平台 API

C# 中你更多使用：

- `Task`
- `async` / `await`
- ThreadPool
- `CancellationToken`
- `Parallel`
- `Channel`
- `IAsyncEnumerable<T>`
- `lock` / `Monitor`
- `SemaphoreSlim`

日常 C# 异步不等于手动创建线程。`async` / `await` 主要是把异步操作写成顺序代码，并由编译器生成状态机。

## 最小 async 示例

```csharp
static async Task<string> DownloadAsync(HttpClient client, string url)
{
    return await client.GetStringAsync(url);
}
```

调用：

```csharp
var html = await DownloadAsync(client, "https://example.com");
```

## `Task` 与线程不是一回事

`Task` 表示一个异步操作或未来结果。它可能：

- 在线程池上运行 CPU 工作。
- 表示正在等待的 I/O。
- 已经完成。
- 被取消。
- 失败并携带异常。

不要把每个 `Task` 都理解成一个独立线程。

## CPU 密集 vs I/O 密集

I/O 密集：

```csharp
var text = await File.ReadAllTextAsync(path);
```

CPU 密集：

```csharp
var result = await Task.Run(() => ExpensiveCalculate(input));
```

ASP.NET Core 中不要随意 `Task.Run` 包业务逻辑。服务器框架本身已经在线程池调度请求，乱用 `Task.Run` 可能只是增加调度成本。

## 取消

```csharp
static async Task RunAsync(CancellationToken cancellationToken)
{
    while (!cancellationToken.IsCancellationRequested)
    {
        await Task.Delay(TimeSpan.FromSeconds(1), cancellationToken);
    }
}
```

取消在 C# 中通常是协作式的。方法需要接收并传递 `CancellationToken`。

## 异常传播

```csharp
try
{
    await DoAsync();
}
catch (InvalidOperationException ex)
{
    logger.LogError(ex, "Failed.");
}
```

异步方法中的异常会存储在返回的 `Task` 中，`await` 时重新抛出。

不要忘记 await：

```csharp
DoAsync(); // 危险：异常可能丢失，操作未被等待
```

## `async void`

除事件处理器外，避免 `async void`：

```csharp
public async void Button_Click(object sender, EventArgs e)
{
    await SaveAsync();
}
```

普通异步方法应返回 `Task` 或 `Task<T>`。

## 并发等待

```csharp
var userTask = GetUserAsync(id);
var ordersTask = GetOrdersAsync(id);

await Task.WhenAll(userTask, ordersTask);

var user = await userTask;
var orders = await ordersTask;
```

这表示并发启动两个异步操作，然后等待都完成。

## `lock`

```csharp
private readonly object _gate = new();
private int _count;

public void Increment()
{
    lock (_gate)
    {
        _count++;
    }
}
```

不要 lock：

- `this`
- `typeof(SomeType)`
- 字符串
- 公开可访问对象

## `SemaphoreSlim`

异步代码中控制并发常用：

```csharp
private readonly SemaphoreSlim _semaphore = new(4);

public async Task ProcessAsync(Item item)
{
    await _semaphore.WaitAsync();
    try
    {
        await HandleAsync(item);
    }
    finally
    {
        _semaphore.Release();
    }
}
```

## `IAsyncEnumerable<T>`

异步流：

```csharp
static async IAsyncEnumerable<int> ReadValuesAsync()
{
    for (var i = 0; i < 10; i++)
    {
        await Task.Delay(100);
        yield return i;
    }
}

await foreach (var value in ReadValuesAsync())
{
    Console.WriteLine(value);
}
```

## 迁移建议

- `async` / `await` 是状态机，不是语法糖级别的小特性。
- I/O 异步优先使用框架提供的 Async API。
- 不要用 `Thread.Sleep` 阻塞异步代码，用 `Task.Delay`。
- 不要用 `.Result` 或 `.Wait()` 阻塞等待异步结果，容易造成死锁或线程池耗尽。
- 共享可变状态仍然需要同步；GC 和托管运行时不会替你解决数据竞争。
