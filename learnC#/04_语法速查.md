# 语法速查

这份速查不追求覆盖所有语法，而是帮你把 C++ 直觉快速映射到 C#。

## 最小程序

C++：

```cpp
#include <iostream>

int main() {
    std::cout << "Hello\n";
}
```

C# 顶级语句：

```csharp
Console.WriteLine("Hello");
```

C# 传统写法：

```csharp
using System;

class Program
{
    static void Main()
    {
        Console.WriteLine("Hello");
    }
}
```

顶级语句适合小程序和示例。大型工程里仍然经常使用显式 `Program` 或 ASP.NET Core 的启动结构。

## 命名空间

C++：

```cpp
namespace app {
class User {};
}
```

C#：

```csharp
namespace App;

public sealed class User
{
}
```

C# 的 file-scoped namespace 可以减少缩进。命名空间不等于程序集；程序集边界由项目构建产物决定。

## 基本类型名称

| C++ | C# |
|---|---|
| `int` | `int` / `System.Int32` |
| `long long` | `long` / `System.Int64` |
| `float` | `float` / `System.Single` |
| `double` | `double` / `System.Double` |
| `bool` | `bool` / `System.Boolean` |
| `char` | `char` / `System.Char`，UTF-16 code unit |
| `std::string` | `string` / `System.String`，不可变引用类型 |

注意：C# 的 `long` 是 64 位；C++ 的 `long` 宽度依平台而变。

## 变量声明

C++：

```cpp
auto count = 10;
const int max = 100;
```

C#：

```csharp
var count = 10;
const int Max = 100;
```

C# 的 `var` 仍然是静态类型，只是让编译器推断局部变量类型。它不是 JavaScript 的 `var`。

## 字符串插值

C++20：

```cpp
// 标准库格式化支持取决于实现
auto text = std::format("id = {}", id);
```

C#：

```csharp
var text = $"id = {id}";
```

C# 字符串不可变，频繁拼接大量字符串时应考虑 `StringBuilder` 或格式化 API。

## 数组和集合

C++：

```cpp
std::vector<int> values{1, 2, 3};
```

C#：

```csharp
int[] array = [1, 2, 3];
List<int> values = [1, 2, 3];
```

C# 数组长度固定；`List<T>` 更像常用的动态数组。

## `if`、`for`、`while`

C# 的基本控制流与 C++ 很像：

```csharp
if (x > 0)
{
    Console.WriteLine(x);
}

for (var i = 0; i < 10; i++)
{
    Console.WriteLine(i);
}

while (queue.Count > 0)
{
    queue.Dequeue();
}
```

C# 条件必须是 `bool`，不能像 C++ 那样把整数隐式当布尔值：

```csharp
if (count)      // 错误
if (count != 0) // 正确
```

## `switch`

C# 的 `switch` 不只是 C 风格分支，还支持模式匹配：

```csharp
var label = value switch
{
    < 0 => "negative",
    0 => "zero",
    > 0 and < 10 => "small",
    _ => "large"
};
```

`_` 是 discard 模式，类似默认分支。

## 函数与方法

C++ 自由函数：

```cpp
int add(int a, int b) {
    return a + b;
}
```

C# 通常把函数放在类型中：

```csharp
static int Add(int a, int b)
{
    return a + b;
}
```

C# 没有全局自由函数。顶级语句里可写局部函数，但工程中常见形式仍然是类、结构体、静态类、扩展方法。

## 属性

C++ 常用 getter/setter：

```cpp
class User {
public:
    int age() const { return age_; }
    void set_age(int value) { age_ = value; }
private:
    int age_{};
};
```

C# 属性：

```csharp
public sealed class User
{
    public int Age { get; set; }
}
```

属性看起来像字段，用起来像字段，但本质是方法。不要在属性 getter 中做昂贵操作或有副作用的逻辑。

## 访问修饰符

| C++ | C# |
|---|---|
| `public` | `public` |
| `protected` | `protected` |
| `private` | `private` |
| 无直接等价 | `internal`，程序集内可见 |
| `friend` | 无直接等价，可用 `InternalsVisibleTo` 等机制 |

C# 顶级类型默认是 `internal`，成员默认通常是 `private`。建议显式写出公共 API 的访问级别。

## `static`

C++ 的 `static` 有多种含义：内部链接、静态存储期、类静态成员。

C# 的 `static` 主要表示：

- 静态字段。
- 静态方法。
- 静态属性。
- 静态类。
- 静态局部函数。

C# 不存在 C++ 文件作用域 `static` 的内部链接语义。

## 注释与文档注释

C# 普通注释：

```csharp
// line
/* block */
```

XML 文档注释：

```csharp
/// <summary>
/// Calculates the total price.
/// </summary>
public decimal CalculateTotal() => 0m;
```

公共库建议写 XML 文档注释，IDE 和文档工具能直接使用。
