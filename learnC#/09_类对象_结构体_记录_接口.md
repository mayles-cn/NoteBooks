# 类、对象、结构体、记录与接口

## 类的基本写法

```csharp
public sealed class User
{
    private readonly string _id;

    public User(string id, string name)
    {
        _id = id;
        Name = name;
    }

    public string Id => _id;
    public string Name { get; private set; }

    public void Rename(string name)
    {
        if (string.IsNullOrWhiteSpace(name))
        {
            throw new ArgumentException("Name cannot be empty.", nameof(name));
        }

        Name = name;
    }
}
```

## 构造函数与初始化

C# 支持构造函数、对象初始化器、主构造函数、`required`、`init` 等多种初始化方式。

```csharp
public sealed class Product
{
    public required string Name { get; init; }
    public decimal Price { get; init; }
}

var product = new Product
{
    Name = "Book",
    Price = 49.9m
};
```

对于有不变量的领域对象，构造函数仍然更合适；对于 DTO 和配置对象，`required` + `init` 很方便。

## 继承

C# 类只支持单继承：

```csharp
public abstract class Shape
{
    public abstract double Area();
}

public sealed class Circle : Shape
{
    public required double Radius { get; init; }

    public override double Area() => Math.PI * Radius * Radius;
}
```

建议默认把类设计成 `sealed`，除非你明确要支持继承扩展。继承是强耦合机制，业务工程中组合和接口通常更稳定。

## 虚方法

C++ 中成员函数默认非虚，写 `virtual` 才虚。C# 中也是类似思路：

```csharp
public class Base
{
    public virtual void Run() {}
}

public class Derived : Base
{
    public override void Run() {}
}
```

如果派生类想隐藏基类成员，需要显式 `new`，否则编译器会警告：

```csharp
public new void Run() {}
```

## 接口

C# 接口是工程中极常用的抽象边界：

```csharp
public interface IClock
{
    DateTimeOffset Now { get; }
}

public sealed class SystemClock : IClock
{
    public DateTimeOffset Now => DateTimeOffset.UtcNow;
}
```

接口适合表达能力和依赖边界。不要为每个类机械创建一个接口；当你需要替换实现、隔离外部资源、测试替身或跨层边界时再引入。

## `record`

`record` 适合不可变数据模型和值语义比较：

```csharp
public sealed record UserDto(string Id, string Name);

var a = new UserDto("1", "Ada");
var b = new UserDto("1", "Ada");
Console.WriteLine(a == b); // True
```

普通 class 默认比较引用身份：

```csharp
var a = new User("1", "Ada");
var b = new User("1", "Ada");
Console.WriteLine(a == b); // 通常 False
```

`record` 很适合 DTO、配置、消息、事件、值对象候选。但带复杂行为和可变状态的实体不一定适合。

## `record struct`

```csharp
public readonly record struct Money(decimal Amount, string Currency);
```

适合小型值对象。注意结构体复制成本和装箱风险。

## 析构函数不是 C++ 析构函数

C# 有 finalizer：

```csharp
public sealed class NativeHandleHolder
{
    ~NativeHandleHolder()
    {
        // finalizer
    }
}
```

这不是 C++ 析构函数：

- 运行时间不确定。
- 由 GC 触发。
- 不应该用于普通托管资源释放。
- 非托管资源应优先使用 `IDisposable` 和 safe handle 模式。

## 对象相等

C# 常见相等层次：

- `ReferenceEquals(a, b)`：是否同一引用。
- `object.Equals(a, b)`：对象相等，可被重写。
- `==`：操作符，可被重载，class 默认通常是引用比较。
- record 默认生成值相等。

如果类型作为字典键或集合元素，需要同时考虑 `Equals` 和 `GetHashCode`。

## 部分类

```csharp
public partial class User
{
}
```

`partial` 允许一个类型分散在多个文件中，常用于：

- 代码生成。
- UI 设计器。
- Source Generator。
- 大型类型按职责拆分。

不要把 `partial` 当作随意拆文件的借口。手写业务代码过度 partial 会降低可读性。

## 迁移建议

- C# 的 `class` 默认引用语义，别把它当 C++ 栈对象。
- C# 的 `struct` 是值类型，谨慎设计为小而不可变。
- `record` 优先考虑数据表达，不要滥用为万能 class。
- 接口用于稳定边界，不是为了“看起来面向对象”。
- `sealed` 是一个好默认值，能减少继承带来的不确定性。
