# 函数、方法、参数传递与扩展方法

## C# 没有自由函数

C++：

```cpp
int add(int a, int b);
```

C# 中方法必须属于某个类型：

```csharp
public static class MathEx
{
    public static int Add(int a, int b) => a + b;
}
```

顶级语句和局部函数能让示例看起来像自由函数，但编译后仍然归属于生成的类型或方法。

## 参数默认按值传递

C#：

```csharp
static void Increment(int x)
{
    x++;
}

var value = 1;
Increment(value);
Console.WriteLine(value); // 仍然是 1
```

对于引用类型，“按值传递”的是引用副本：

```csharp
static void Rename(User user)
{
    user.Name = "new";
}
```

这里没有改变调用方变量本身，但改变了它指向对象的内部状态。

## `ref`、`out`、`in`

### `ref`

```csharp
static void Increment(ref int x)
{
    x++;
}

var value = 1;
Increment(ref value);
```

`ref` 表示按引用传递，调用点也必须写 `ref`，这能让副作用更显眼。

### `out`

```csharp
static bool TryParseAge(string text, out int age)
{
    return int.TryParse(text, out age);
}
```

`out` 常用于 Try 模式。被调用方法必须在返回前赋值。

### `in`

```csharp
static decimal Calculate(in Money money)
{
    return money.Amount;
}
```

`in` 表示只读引用传递，适合较大的只读结构体。不要为了“看起来高性能”滥用，先测量。

## 可选参数与命名参数

```csharp
static void Connect(string host, int port = 443, bool useTls = true)
{
}

Connect("example.com", useTls: false);
```

公共 API 的可选参数要谨慎，因为默认值会编译进调用方。库升级时可能出现调用方仍使用旧默认值的问题。

## 方法重载

C# 支持重载：

```csharp
void Log(string message) {}
void Log(Exception exception) {}
```

但 C# 没有 C++ 的函数模板重载解析复杂度。泛型约束和 nullable 会影响可读性，API 设计时应避免过多相似重载。

## 表达式体成员

```csharp
public int Count => _items.Count;

public bool IsEmpty() => _items.Count == 0;
```

适合短逻辑。复杂逻辑不要硬压成一行。

## 局部函数

```csharp
int SumPositive(IEnumerable<int> values)
{
    var sum = 0;

    foreach (var value in values)
    {
        if (IsPositive(value))
        {
            sum += value;
        }
    }

    return sum;

    static bool IsPositive(int value) => value > 0;
}
```

局部函数适合把只服务当前方法的小逻辑收进去。能声明 `static` 时优先声明，避免无意捕获外部变量。

## Lambda

C++：

```cpp
auto add = [](int a, int b) { return a + b; };
```

C#：

```csharp
Func<int, int, int> add = (a, b) => a + b;
```

常见委托类型：

- `Action`：无返回值。
- `Action<T>`：一个参数，无返回值。
- `Func<TResult>`：有返回值。
- `Func<T, TResult>`：一个参数，有返回值。
- `Predicate<T>`：返回 bool 的谓词。

## 扩展方法

扩展方法让你像给已有类型添加实例方法：

```csharp
public static class StringExtensions
{
    public static bool IsBlank(this string? value)
    {
        return string.IsNullOrWhiteSpace(value);
    }
}
```

调用：

```csharp
if (name.IsBlank())
{
}
```

扩展方法只是静态方法的语法糖，不会真的修改原类型。适合给领域代码增加可读性，但滥用会让 API 来源不清晰。

## 委托

```csharp
public delegate void ProgressChanged(int percent);

public sealed class Downloader
{
    public event ProgressChanged? ProgressChanged;
}
```

委托是类型安全的函数引用。事件基于委托，但加入了订阅/取消订阅语义。

## 迁移建议

- C++ 喜欢用自由函数表达无状态算法；C# 常用静态类、扩展方法或服务类承载。
- 引用类型参数默认不是“深拷贝”，方法内修改对象状态会影响调用方。
- `out` 适合 Try 模式，但复杂返回值优先考虑对象、元组或结果类型。
- Lambda 捕获会产生闭包，性能敏感路径要留意分配。
