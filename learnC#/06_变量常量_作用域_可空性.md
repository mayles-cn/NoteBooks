# 变量、常量、作用域与可空性

## 局部变量必须明确赋值

C++：

```cpp
int x;
std::cout << x; // 未定义行为
```

C#：

```csharp
int x;
Console.WriteLine(x); // 编译错误：使用前未赋值
```

C# 编译器会进行 definite assignment 分析，防止读取未初始化的局部变量。

字段则会有默认值：

```csharp
public sealed class Counter
{
    private int _count;       // 默认 0
    private string? _name;    // 默认 null
}
```

## `const` 与 `readonly`

C# 有两个常被 C++ 程序员混淆的关键字。

### `const`

```csharp
public const int MaxCount = 100;
```

特点：

- 编译期常量。
- 只能用于内置数值、`bool`、`char`、`string`、`null` 等有限类型。
- 跨程序集使用时值可能被编译进调用方，公共常量变更要谨慎。

### `readonly`

```csharp
public sealed class Settings
{
    public readonly string Path;

    public Settings(string path)
    {
        Path = path;
    }
}
```

特点：

- 运行期只读字段。
- 可以在声明处或构造函数中赋值。
- 更适合公共库中的常量式配置。

## `static readonly`

常见写法：

```csharp
public static readonly TimeSpan DefaultTimeout = TimeSpan.FromSeconds(30);
```

它适合非编译期常量，例如对象、结构体、数组引用等。但注意：`readonly` 限制字段引用不能改，不代表引用对象内部一定不可变。

## `var` 的边界

推荐：

```csharp
var users = new List<User>();
var total = price * count;
```

不推荐：

```csharp
var data = Get(); // 看不出类型时降低可读性
```

`var` 是静态类型推断，不会让变量变成动态类型。

## 作用域

C# 块级作用域与 C++ 类似：

```csharp
if (enabled)
{
    var message = "ok";
}

// message 在这里不可见
```

但 C# 的 pattern variable 有自己的作用域规则：

```csharp
if (obj is string s)
{
    Console.WriteLine(s.Length);
}
```

变量 `s` 在模式匹配成功的分支内可用。

## 默认值

```csharp
default(int)       // 0
default(bool)      // false
default(string)    // null
default(DateTime)  // 0001-01-01
```

泛型代码里常见：

```csharp
T value = default!;
```

`!` 是 null-forgiving operator，告诉编译器“这里我知道不是 null 或我会负责”。它只影响警告，不改变运行时行为。

## `required`

现代 C# 可以声明对象初始化时必须设置的属性：

```csharp
public sealed class User
{
    public required string Name { get; init; }
    public int Age { get; init; }
}

var user = new User { Name = "Ada", Age = 36 };
```

这比写很多构造函数更适合 DTO、配置对象、请求模型。

## `init`

`init` 表示对象初始化后不可再设置：

```csharp
public sealed class Order
{
    public string Id { get; init; } = "";
}

var order = new Order { Id = "A001" };
// order.Id = "A002"; // 错误
```

它不是深不可变。属性指向的对象如果本身可变，内部仍可变。

## 可空性习惯

建议启用：

```xml
<Nullable>enable</Nullable>
```

然后形成习惯：

```csharp
string name;       // 不应为 null
string? nickName;  // 允许为 null
```

处理可空值：

```csharp
if (user.Name is not null)
{
    Console.WriteLine(user.Name.Length);
}
```

空合并：

```csharp
var displayName = user.NickName ?? user.Name ?? "unknown";
```

空条件：

```csharp
var length = user.NickName?.Length;
```

## C++ 习惯迁移建议

- 不要用 `null` 表示所有失败，优先考虑异常、`bool TryX(out T value)`、结果类型。
- 不要滥用 `!` 压制可空警告。
- `readonly` 不等于 C++ 的 `const` 成员函数体系。
- 可变共享状态要谨慎，C# 引用类型默认共享引用。
- 小心静态可变字段，它们在测试和并发中很容易造成隐蔽问题。
