# 表达式、控制流与模式匹配

## 条件表达式必须是 bool

C++：

```cpp
if (ptr) {}
if (count) {}
```

C#：

```csharp
if (obj is not null) {}
if (count != 0) {}
```

C# 不允许整数、引用直接作为布尔条件。这能减少一类隐式转换错误。

## 三元表达式

```csharp
var label = score >= 60 ? "pass" : "fail";
```

与 C++ 类似，但 C# 的目标类型推断、nullable 分析、模式匹配会参与类型判断。

## `switch` 语句

```csharp
switch (command)
{
    case "start":
        Start();
        break;
    case "stop":
        Stop();
        break;
    default:
        throw new InvalidOperationException("Unknown command.");
}
```

C# 不允许普通 `case` 隐式 fall through 到下一个 `case`。如果要跳转，需要显式 `goto case`，但实际很少这么写。

## `switch` 表达式

```csharp
var price = level switch
{
    "vip" => 80m,
    "normal" => 100m,
    _ => throw new ArgumentOutOfRangeException(nameof(level))
};
```

它适合把输入映射为输出。比写很多 `if` 更清晰，也更容易保持穷尽性思维。

## 模式匹配

### 类型模式

```csharp
if (shape is Circle circle)
{
    Console.WriteLine(circle.Radius);
}
```

相当于“判断类型 + 安全转换 + 声明变量”。

### 属性模式

```csharp
if (user is { Age: >= 18, IsActive: true })
{
    Console.WriteLine("adult active user");
}
```

### 关系模式

```csharp
var level = score switch
{
    >= 90 => "A",
    >= 80 => "B",
    >= 60 => "C",
    _ => "D"
};
```

### 组合模式

```csharp
if (value is >= 0 and <= 100)
{
    Console.WriteLine("valid");
}
```

## `foreach`

C++ range-for：

```cpp
for (const auto& item : items) {
}
```

C#：

```csharp
foreach (var item in items)
{
}
```

C# 的 `foreach` 基于枚举器模式，常见集合都支持。注意枚举期间修改集合通常会抛异常。

## `yield return`

C# 可以很方便地写迭代器：

```csharp
static IEnumerable<int> Range(int start, int count)
{
    for (var i = 0; i < count; i++)
    {
        yield return start + i;
    }
}
```

它会由编译器生成状态机。这个机制和 C++20 coroutine 有概念上的相似处，但语法和生态用法不同。

## `using` 语句与 `using` 声明

`using` 在 C# 有多种含义。资源释放场景：

```csharp
using var file = File.OpenRead("data.txt");
// 离开作用域时自动 Dispose
```

传统块形式：

```csharp
using (var file = File.OpenRead("data.txt"))
{
    // use file
}
```

这不是 C++ RAII 的完整等价物，但在非托管资源释放上承担类似职责。

## `checked` 与 `unchecked`

C# 整数溢出检查可控制：

```csharp
checked
{
    int x = int.MaxValue;
    x++;
}
```

默认是否检查取决于上下文和编译选项。金融、计数等敏感逻辑建议明确使用 `checked` 或测试覆盖边界。

## 空传播与空合并

```csharp
var city = user.Address?.City ?? "unknown";
```

这比层层 `if (x != null)` 更简洁。但不要因为语法简洁就放松领域建模；如果某个对象按业务规则必须存在，应该在边界处校验。

## 控制流迁移建议

- C# 的 `switch` 表达式和模式匹配很强，适合替代部分 visitor、类型判断和状态映射。
- 不要用异常控制普通分支。
- `foreach` 清晰但可能隐藏枚举器和分配成本，性能敏感处要测量。
- `yield return` 适合流式枚举，但调试状态机时要知道它是延迟执行。
