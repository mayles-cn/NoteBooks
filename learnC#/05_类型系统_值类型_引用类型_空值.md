# 类型系统、值类型、引用类型与空值

## 最大差异：值类型与引用类型

C# 类型大体分为两类：

- 值类型：`struct`、`enum`、内置数值类型、`bool`、`char`、`DateTime` 等。
- 引用类型：`class`、`interface`、`delegate`、`string`、数组等。

这比 C++ 的“对象可以在栈上、堆上、静态区上”更偏语言层。C# 的 `class` 实例通常通过引用访问，`struct` 值通常按值复制。

## 对比示例

C++：

```cpp
struct Point {
    int x;
    int y;
};

Point a{1, 2};
Point b = a; // 复制
b.x = 10;    // a 不变
```

C# `struct`：

```csharp
public struct Point
{
    public int X;
    public int Y;
}

var a = new Point { X = 1, Y = 2 };
var b = a; // 复制
b.X = 10;  // a 不变
```

C# `class`：

```csharp
public sealed class User
{
    public int Age { get; set; }
}

var a = new User { Age = 20 };
var b = a; // 复制引用
b.Age = 30; // a.Age 也变为 30
```

## C# 的 `struct` 不等于 C++ 的 `struct`

| 维度 | C++ `struct` | C# `struct` |
|---|---|---|
| 默认访问级别 | public | 成员默认 private |
| 类型类别 | 与 class 几乎只差默认访问/继承默认值 | 值类型 |
| 继承 | 可继承，可有虚函数 | 不能继承其他 struct/class，但可实现接口 |
| 构造/析构 | 可自定义复杂生命周期 | 无 C++ 析构语义，生命周期由值语义和 GC 环境决定 |
| 内存控制 | 可高度控制布局 | 默认由运行时管理，可用特性影响布局 |

C# 的 `struct` 应该用于小型、不可变、值语义明确的类型，例如坐标、范围、金额值对象。不要把大型可变对象做成 `struct`。

## 装箱与拆箱

值类型转换为 `object` 或接口时可能发生装箱：

```csharp
int x = 42;
object boxed = x;    // 装箱：复制到托管堆对象
int y = (int)boxed;  // 拆箱
```

装箱会产生分配和复制成本。对于性能敏感代码，要注意：

- 把值类型放入非泛型集合会装箱。
- 通过接口调用值类型方法可能装箱，具体取决于上下文。
- 使用泛型集合 `List<int>` 通常可避免装箱。

## `string` 是引用类型，但不可变

C++：

```cpp
std::string name = "abc";
name[0] = 'A';
```

C#：

```csharp
string name = "abc";
// name[0] = 'A'; // 错误，string 不可变
name = "Abc";     // 重新指向新字符串
```

不要因为 `string` 使用起来像值就误以为它是值类型。它是引用类型，只是不可变，很多 API 也按值语义设计。

## 可空值类型

普通值类型不能为 `null`：

```csharp
int x = null; // 错误
```

可空值类型：

```csharp
int? maybe = null;

if (maybe.HasValue)
{
    Console.WriteLine(maybe.Value);
}
```

`int?` 是 `Nullable<int>` 的简写。

## 可空引用类型

现代 C# 项目通常启用：

```xml
<Nullable>enable</Nullable>
```

启用后：

```csharp
string name = null;  // 警告
string? text = null; // 允许
```

注意：可空引用类型主要是编译器静态分析，不是运行时创建了两种不同引用类型。`string` 和 `string?` 在运行时本质都是 `System.String` 引用。

## C++ 指针/引用与 C# 引用

| 概念 | C++ | C# |
|---|---|---|
| 可空指针 | `T*` 可为 `nullptr` | 引用类型可为 `null`，可空分析帮助检查 |
| 引用 | `T&` 通常必须绑定对象 | C# 引用类型变量可重新赋值、可为 null |
| 原始指针 | 核心语言能力 | 需要 `unsafe` 上下文 |
| 对象地址稳定性 | 可控 | GC 可能移动对象，需 pinning |

C# 的引用不是 C++ 的引用。它更接近“受运行时管理的对象句柄”。

## 类型转换

C# 数值转换比 C++ 更保守：

```csharp
int x = 10;
long y = x;       // 隐式安全转换
int z = (int)y;   // 显式转换
```

引用类型转换：

```csharp
object obj = "hello";

if (obj is string s)
{
    Console.WriteLine(s.Length);
}
```

`is` 模式匹配比直接强转更安全。

## `dynamic`

C# 有 `dynamic`：

```csharp
dynamic x = "hello";
Console.WriteLine(x.Length);
```

它会把部分绑定推迟到运行时。除非做 COM、动态 JSON、脚本互操作等场景，否则不要把 `dynamic` 当成日常工具。
